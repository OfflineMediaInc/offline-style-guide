Page Structure

Different page types require different structural approaches. Understanding when to use borders vs shadows is critical for consistent styling.

Page Types

Landing Pages
  • Full-width hero sections
  • Large display typography
  • Minimal borders—content flows freely
  • Shadows only on floating CTAs
  • High visual impact
Detail Pages
  • Two-column layout
  • Information-dense sections
  • Borders for content containment
  • Minimal shadow usage
  • Higher information density

Borders vs Shadows: When to Use Each

Key Principle: Shadows indicate that an element is "floating above" the page. Static content doesn't float—use borders for containment, shadows for elevated elements.

✓ CORRECT: Border for containment
Restaurant Hours
Mon-Thu: 11am - 9pm
Fri-Sat: 11am - 10pm
Sun: 12pm - 8pm
Static info block uses border only
✗ INCORRECT: Border AND shadow
Restaurant Hours
Mon-Thu: 11am - 9pm
Fri-Sat: 11am - 10pm
Sun: 12pm - 8pm
Excessive—choose border OR shadow, not both
✓ CORRECT: Shadow for floating CTA
$30 to spend
Sticky footer CTA needs elevation
✓ CORRECT: Spacing without decoration
About This Restaurant
A neighborhood wine bar serving small plates and an extensive wine list. Perfect for date night or catching up with friends.
Flowing content needs no containment

Shadow Usage Reference

Element Type Recommendation
Static content cards No shadow — use border instead
Bordered info sections No shadow — border provides containment
Sticky footer/header --card-shadow — needs elevation
Modal dialogs --new-shadow — high elevation
Dropdown menus --card-shadow or --new-shadow
Floating action buttons --card-shadow
Popovers/tooltips --new-shadow
